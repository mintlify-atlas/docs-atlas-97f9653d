---
title: "Assign Subscription"
api: "POST /sub/assign"
description: "Assign a subscription to any user (Admin only)"
---

Allows administrators to assign or update a subscription for any user in the system. This is useful for customer support, promotional grants, or manual subscription management.

<Warning>
  This endpoint requires admin access. Only users with the `admin` role can assign subscriptions to other users.
</Warning>

## Authentication

Requires a valid JWT token with admin privileges.

```bash
Authorization: Bearer <admin_token>
```

## Request Body

<ParamField body="userId" type="string" required>
  The ID of the user to assign the subscription to
</ParamField>

<ParamField body="subscriptionTypeId" type="string" required>
  The name of the subscription plan to assign. Must be one of: `trial`, `monthly`, or `annual`
</ParamField>

## Behavior

- If the user already has a subscription (active or inactive), it will be updated with the new subscription type
- If the user doesn't have any subscription record, a new one will be created
- The `activatedBy` field will be set to the admin user's ID who performed the assignment
- Any existing subscription will be replaced, and the new subscription period starts immediately

## Use Cases

- **Customer Support**: Extend or upgrade a user's subscription plan
- **Promotional Grants**: Provide free trials or complimentary subscriptions
- **Manual Corrections**: Fix subscription issues or billing problems
- **Testing**: Set up test accounts with specific subscription levels

## How It Affects Game Access

When an admin assigns a subscription to a user:
1. The user immediately gains access to all subscription-based games
2. They can add subscription games to their library
3. Access continues until the `endDate` is reached
4. The subscription can be tracked back to the admin who assigned it via the `activatedBy` field
