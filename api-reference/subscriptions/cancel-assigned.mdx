---
title: "Cancel Assigned Subscription"
api: "POST /sub/cancelAssign"
description: "Cancel another user's subscription (Admin or authorized user)"
---

Allows authorized users to cancel a subscription for a specified user. While the route doesn't explicitly require admin privileges in the middleware, it's designed for administrative use and customer support scenarios.

## Authentication

Requires a valid JWT token in the Authorization header.

```bash
Authorization: Bearer <token>
```

## Request Body

<ParamField body="userId" type="string" required>
  The ID of the user whose subscription should be cancelled
</ParamField>

## Behavior

- Finds the active subscription for the specified user
- Sets the subscription's `isActive` field to `false`
- Updates the `endDate` to the current timestamp
- User immediately loses access to subscription-based games
- Does not affect purchased games

## Comparison with Cancel Own Subscription

| Feature | `/sub/cancel` | `/sub/cancelAssign` |
|---------|---------------|---------------------|
| Target User | Authenticated user (self) | Any user specified by ID |
| Request Body | None required | Requires `userId` |
| Use Case | User self-service | Admin/support operations |
| Authorization | Any authenticated user | Typically admin access |

## Example Scenario

An admin needs to cancel a user's subscription due to a billing dispute:

1. User with ID `60f6c5b7f58a5c6d09fa1a28` has an annual subscription
2. Subscription started on February 15, 2026
3. Original end date was February 15, 2027
4. Admin cancels on March 5, 2026 at 3:45 PM
5. New end date becomes March 5, 2026 at 3:45 PM
6. User loses access immediately

## Impact on Game Access

When a subscription is cancelled via this endpoint:

- **Immediate Effect**: User loses access to all subscription-based games
- **Library Impact**: Subscription games remain in library but are unplayable
- **Purchase Protection**: Individually purchased games remain fully accessible
- **Progress Retention**: All game saves and progress data is preserved

## Important Notes

<Warning>
  This endpoint allows cancellation of any user's subscription. Implement proper authorization checks in your application logic to ensure only authorized personnel can use this endpoint.
</Warning>

- The error message "No tienes una suscripción activa" technically refers to the target user, not the requester
- This endpoint only affects active subscriptions - inactive subscriptions cannot be cancelled
- Consider implementing audit logging to track who cancelled which subscriptions
- Unlike the assignment endpoint, this does NOT require explicit admin middleware (though it should in production)
