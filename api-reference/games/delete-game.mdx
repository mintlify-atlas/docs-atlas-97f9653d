---
title: 'Delete Game'
api: 'DELETE /game/delete/{videojuegoId}'
description: 'Delete a game from the catalog (Admin only)'
---

Permanently removes a video game from the system.

<Warning>
  This endpoint is restricted to administrators only. Regular users will receive a 403 Forbidden response. This action is irreversible.
</Warning>

## Authentication

Requires JWT authentication token with admin role:

```
Authorization: Bearer <admin-token>
```

## Path Parameters

<ParamField path="videojuegoId" type="string" required>
  The unique identifier of the video game to delete
</ParamField>

## Response

On successful deletion, returns HTTP status code 204 (No Content) with no response body.

## Implementation Details

Implemented in `src/controllers/videojuego.controller.js:152-160` and `src/routes/videojuego.router.js:9`:

### Deletion Process
1. Validates the videojuegoId parameter
2. Calls `videojuegoService.deleteVideojuegoById(videojuegoId)`
3. Returns 204 status with no content on success

### Data Cleanup
- The game document is removed from the database
- **Image file is NOT automatically deleted** from `/public/images/`
- Related library entries may become orphaned (consider cleanup)

## Authorization

Route protection: `protect` → `restrictToAdmin` middleware chain

- `protect`: Validates JWT token and identifies user
- `restrictToAdmin`: Checks if user role is "admin"
- Non-admin users receive 403 Forbidden response

## Side Effects

<Warning>
  Deleting a game may have the following side effects:
</Warning>

- **Library entries**: Users who added this game to their library will have orphaned references
- **Image files**: The cover image file remains on disk in `/public/images/` directory
- **Statistics**: Any analytics or play history tied to this game may become invalid

## Best Practices

Before deleting a game:

1. **Check for dependencies**: Verify if the game is in any user's library
2. **Backup data**: Consider archiving game data before deletion
3. **Notify users**: If users have this game in their library, consider notifying them
4. **Clean up files**: Manually delete the associated image file if needed

## Error Responses

- `401 Unauthorized`: Missing or invalid authentication token
- `403 Forbidden`: User is not an administrator
- `404 Not Found`: Game with specified ID does not exist

## Notes

- This is a hard delete operation - the game is permanently removed from the database
- There is no soft-delete or archive functionality
- The operation returns 204 No Content on success (no response body)
- Consider implementing a soft-delete with an `isActive` flag for production systems
