---
title: 'Update Game'
api: 'PATCH /game/update/{videojuegoId}'
description: 'Update an existing game (Admin only)'
---

Updates an existing video game in the system. This endpoint accepts multipart/form-data to support updating the game's cover image.

<Warning>
  This endpoint is restricted to administrators only. Regular users will receive a 403 Forbidden response.
</Warning>

## Authentication

Requires JWT authentication token with admin role:

```
Authorization: Bearer <admin-token>
```

## Path Parameters

<ParamField path="videojuegoId" type="string" required>
  The unique identifier of the video game to update
</ParamField>

## Request Body

Content-Type: `multipart/form-data`

All fields are optional - only include fields you want to update:

<ParamField body="title" type="string">
  Updated title of the video game
</ParamField>

<ParamField body="genre" type="string">
  Updated genre of the game
</ParamField>

<ParamField body="description" type="string">
  Updated description of the game
</ParamField>

<ParamField body="releaseDate" type="string">
  Updated release date in YYYY-MM-DD format
</ParamField>

<ParamField body="subscription" type="boolean">
  Whether the game is available through subscription
</ParamField>

<ParamField body="purchase" type="boolean">
  Whether the game can be purchased
</ParamField>

<ParamField body="price" type="number">
  Updated price of the game
</ParamField>

<ParamField body="image" type="file">
  New cover image file for the game (optional)
</ParamField>

## Implementation Details

Implemented in `src/controllers/videojuego.controller.js:138-151` and `src/routes/videojuego.router.js:8`:

### Partial Updates
- Only fields included in the request are updated
- Existing values are preserved for fields not in the request
- Uses Mongoose's `findByIdAndUpdate` with the update data

### Image Update
- Image update is **optional**
- If a new image is uploaded:
  - New file is saved to `/public/images/`
  - `imagePath` is updated with new path
  - Old image file remains on disk (not automatically deleted)
- If no image in request, existing image path is unchanged

### Access Type Updates
- Can update `subscription` and `purchase` independently
- Boolean conversion handled if sent as strings
- Price can be updated separately from access type

## Authorization

Route protection: `protect` → `restrictToAdmin` middleware chain

- Only administrators can update games
- Non-admin users receive 403 Forbidden response

## Notes

- The update is a partial update (PATCH), not a full replacement
- You can update just the title, just the price, or any combination of fields
- Image updates replace the path but don't clean up old files
- All validation rules from the Videojuego model still apply
