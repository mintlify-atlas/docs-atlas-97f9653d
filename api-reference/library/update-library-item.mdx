---
title: "Update Library Item"
api: "PATCH /library/{bibliotecaId}"
description: "Update the access type of a game in the user's library"
---

Allows users to update how they access a specific game in their library, such as upgrading from subscription access to purchased access.

## Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
  
  Example: `Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...`
</ParamField>

## Path Parameters

<ParamField path="bibliotecaId" type="string" required>
  The unique identifier of the library entry to update
  
  Example: `"60f6c5b7f58a5c6d09fa1a28"`
</ParamField>

## Request Body

<ParamField body="accessType" type="string" required>
  The new access type for this game
  
  **Options:**
  - `subscription` - Game accessed through active subscription
  - `purchase` - Game purchased permanently
  
  Example: `"purchase"`
</ParamField>

## Common Use Cases

### Upgrade from Subscription to Purchase
Users who initially accessed a game through subscription may want to purchase it permanently:

```json
{
  "accessType": "purchase"
}
```

This ensures they retain access even if their subscription expires.

### Downgrade from Purchase to Subscription
While less common, users might want to change their access method:

```json
{
  "accessType": "subscription"
}
```

<Warning>
  Changing from `purchase` to `subscription` means the user will lose access if their subscription expires or is cancelled.
</Warning>

## Access Type Comparison

| Feature | Purchase | Subscription |
|---------|----------|-------------|
| **Permanence** | Permanent | Depends on active subscription |
| **Cost** | One-time payment | Recurring fee |
| **Access** | Lifetime | While subscribed |
| **Transferability** | Non-transferable | Non-transferable |

## Validation

- The `accessType` must be either `"purchase"` or `"subscription"`
- The library entry must belong to the authenticated user
- The library entry must exist

## Implementation Notes

- Controller: `src/controllers/biblioteca.controller.js:67` (`actualizarJuegoDeBiblioteca`)
- Route: `PATCH /library/:bibliotecaId` (protected)
- Only the access type can be updated; other fields remain unchanged
- The `addedAt` timestamp is preserved from the original entry
