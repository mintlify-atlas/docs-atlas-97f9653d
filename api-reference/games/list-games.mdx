---
title: 'List Games'
api: 'GET /game'
description: 'Get all games with filtering based on subscription access'
---

Returns all video games available in the system. The results are filtered based on the authenticated user's subscription status:

- **Users with active subscription**: See all games (subscription-based and purchasable)
- **Users without subscription**: Only see games available for purchase
- **Admins**: See all games regardless of subscription status

## Authentication

Requires JWT authentication token in the Authorization header:

```
Authorization: Bearer <token>
```

## Request

No request parameters required.

## Filtering Logic

The endpoint implements subscription-based filtering in `src/controllers/videojuego.controller.js:118-136`:

1. **Admin users**: No filtering applied - see all games
2. **Non-subscribed users**: Only games with `accessType.purchase === true` are returned
3. **Subscribed users**: All games are returned

## Notes

- The subscription status is checked using the `SubscribedUser` model
- Games with `accessType.subscription = true` and `accessType.purchase = false` are hidden from non-subscribers
- Games available for purchase (`accessType.purchase = true`) are always visible to all authenticated users
