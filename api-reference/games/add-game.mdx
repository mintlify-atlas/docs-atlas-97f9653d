---
title: 'Add Game'
api: 'POST /game/add'
description: 'Add a new game to the catalog (Admin only)'
---

Creates a new video game in the system. This endpoint accepts multipart/form-data to support image uploads.

<Warning>
  This endpoint is restricted to administrators only. Regular users will receive a 403 Forbidden response.
</Warning>

## Authentication

Requires JWT authentication token with admin role:

```
Authorization: Bearer <admin-token>
```

## Request Body

Content-Type: `multipart/form-data`

<ParamField body="title" type="string" required>
  The title of the video game
</ParamField>

<ParamField body="genre" type="string" required>
  The genre of the game (e.g., "Action", "Adventure", "RPG")
</ParamField>

<ParamField body="description" type="string" required>
  Detailed description of the game
</ParamField>

<ParamField body="releaseDate" type="string" required>
  Release date in YYYY-MM-DD format
</ParamField>

<ParamField body="subscription" type="boolean" required>
  Whether the game is available through subscription (send as string "true" or "false")
</ParamField>

<ParamField body="purchase" type="boolean" required>
  Whether the game can be purchased (send as string "true" or "false")
</ParamField>

<ParamField body="price" type="number" required>
  Price of the game (set to 0 for subscription-only games)
</ParamField>

<ParamField body="image" type="file" required>
  Cover image file for the game (uploaded as multipart file)
</ParamField>

## Implementation Details

Implemented in `src/controllers/videojuego.controller.js:6-31` and `src/routes/videojuego.router.js:7`:

### Image Upload
- Uses multer middleware (`upload.single('image')`)
- Image is stored in `/public/images/` directory
- Filename is auto-generated by multer
- Path is saved as `/public/images/{filename}`

### Access Type Conversion
- Boolean values are sent as strings in multipart/form-data
- Controller converts `"true"`/`"false"` strings to actual booleans
- `accessType` object is constructed from `subscription` and `purchase` fields

### Price Handling
- If `purchase` is false, price is set to 0 regardless of input
- Price is only relevant for purchasable games

## Validation

- Image upload is **required** - returns 400 if missing
- All text fields are validated by the Videojuego model schema
- Admin role is enforced by `restrictToAdmin` middleware

## Authorization

Route protection: `protect` → `restrictToAdmin` middleware chain

- `protect`: Validates JWT token and identifies user
- `restrictToAdmin`: Checks if user role is "admin"
