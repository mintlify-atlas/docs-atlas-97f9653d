---
title: "Get Library Item"
api: "GET /library/{bibliotecaId}"
description: "Retrieve details of a specific game in the user's library"
---

Returns detailed information about a specific game in the authenticated user's library.

## Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
  
  Example: `Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...`
</ParamField>

## Path Parameters

<ParamField path="bibliotecaId" type="string" required>
  The unique identifier of the library entry (not the game ID)
  
  Example: `"60f6c5b7f58a5c6d09fa1a28"`
</ParamField>

## Response

<ResponseField name="status" type="string">
  Response status indicator
</ResponseField>

<ResponseField name="data" type="object">
  Container for the response data
  
  <Expandable title="data object">
    <ResponseField name="juego" type="object">
      The library entry with populated game details
      
      <Expandable title="Library Entry">
        <ResponseField name="_id" type="string">
          Unique identifier for the library entry
        </ResponseField>
        
        <ResponseField name="user" type="string">
          User ID who owns this library entry
        </ResponseField>
        
        <ResponseField name="videojuego" type="object">
          Full game object with all details
          
          <Expandable title="Game Details">
            <ResponseField name="_id" type="string">
              Game identifier
            </ResponseField>
            
            <ResponseField name="title" type="string">
              Game title
            </ResponseField>
            
            <ResponseField name="genre" type="string">
              Game genre
            </ResponseField>
            
            <ResponseField name="description" type="string">
              Game description
            </ResponseField>
            
            <ResponseField name="releaseDate" type="string">
              Release date in ISO 8601 format
            </ResponseField>
            
            <ResponseField name="imagePath" type="string">
              Path to game cover image
            </ResponseField>
            
            <ResponseField name="accessType" type="object">
              Defines how the game can be accessed
              
              <ResponseField name="subscription" type="boolean">
                Whether game is available via subscription
              </ResponseField>
              
              <ResponseField name="purchase" type="boolean">
                Whether game can be purchased
              </ResponseField>
            </ResponseField>
            
            <ResponseField name="price" type="number">
              Game purchase price
            </ResponseField>
          </Expandable>
        </ResponseField>
        
        <ResponseField name="accessType" type="string">
          How the user acquired access to this game
          
          - `purchase` - User purchased the game
          - `subscription` - User has access through active subscription
        </ResponseField>
        
        <ResponseField name="addedAt" type="string">
          Timestamp when game was added to library (ISO 8601 format)
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

## Use Cases

- Display detailed view of a game in user's library
- Check access type for a specific library entry
- Retrieve full game metadata for a library item
- Verify when a game was added to the library

## Important Notes

<Note>
  The `bibliotecaId` parameter refers to the **library entry ID**, not the game ID. Each user can have only one library entry per game, but the library entry ID is unique to that user-game combination.
</Note>

## Implementation Notes

- Controller: `src/controllers/biblioteca.controller.js:55` (`obtenerJuegoDeBiblioteca`)
- Route: `GET /library/:bibliotecaId` (protected)
- The videojuego field is populated with full game details
- Only the authenticated user can access their own library entries
