---
title: "Add to Library"
api: "POST /library/add"
description: "Add a game to the user's library with specified access type"
---

Allows an authenticated user to add a game to their library. The game can be added either through purchase or subscription access.

## Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
  
  Example: `Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...`
</ParamField>

## Request Body

<ParamField body="videojuegoId" type="string" required>
  The unique identifier of the game to add to the library
  
  Example: `"60f6c5b7f58a5c6d09fa1a28"`
</ParamField>

<ParamField body="accessType" type="string" required>
  The type of access for this game
  
  **Options:**
  - `subscription` - Game accessed through active subscription
  - `purchase` - Game purchased permanently
  
  Example: `"subscription"`
</ParamField>

## Access Control

### Subscription Access
When adding a game with `accessType: "subscription"`:
- User must have an active subscription
- Trial subscription users cannot add subscription games
- Access is tied to subscription status

### Purchase Access
When adding a game with `accessType: "purchase"`:
- User permanently owns the game
- No subscription required
- Access is permanent regardless of subscription status

## Validation Rules

1. **Duplicate Prevention**: Users cannot add the same game twice, even with different access types
2. **Subscription Validation**: Trial users are blocked from adding subscription games
3. **Access Type Enforcement**: The access type must match one of the game's available access methods

## Error Codes

| Status | Description |
|--------|-------------|
| 201 | Game successfully added to library |
| 400 | Game already exists in library |
| 403 | User lacks required subscription for subscription access |
| 404 | Game not found |

## Implementation Notes

- Controller: `src/controllers/biblioteca.controller.js:6` (`agregarJuegoABiblioteca`)
- Route: `POST /library/add` (protected)
- The endpoint checks subscription status before allowing subscription-based game additions
- Duplicate entries are prevented at the application level
